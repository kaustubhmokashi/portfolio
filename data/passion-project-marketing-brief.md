# Passion Project Marketing Brief

This brief is for the four native tools on the site:

1. PocketEncode
2. DriveLink Direct
3. WebP Forge
4. ToneSplit Posterizer

Use the actual UI from the corresponding HTML pages as the visual reference.
Do not use screenshots as the source of truth. Recreate the UI composition from code,
then turn each piece into a polished JPG marketing asset.

Required output per project:
- 1 thumbnail image at `1600x1125`
- 1 hero image at `2400x2000`
- 1 short description, about 20 words
- 1 detailed description, about 200 words, written from a UX designer's point of view

The thumbnail should feel like the product at work.
The hero should feel like a more cinematic marketing image, usually with a person using the product.

---

## 1) PocketEncode

**Product positioning**  
Browser MP4 compression with precise controls for size, quality, and delivery.

**Short Description**  
A browser MP4 compressor that helps creators balance file size, quality, and delivery with clear controls and instant feedback.

**Long Description**  
PocketEncode is designed around a simple UX truth: people do not want to think about compression, they want to finish it. I approached this tool by reducing the distance between “I need this video smaller” and “I have a file that is ready to ship.” The interface makes every important decision visible, from size targets to quality tradeoffs, so the user always knows what the product is doing and why.

As a UX designer, I wanted the workflow to feel calm and predictable. Video tools often overwhelm users with technical language, but PocketEncode keeps the interaction anchored in a few meaningful controls. The preview, compression settings, and output feedback work together to turn an anxious task into a guided one. The user can move a control and immediately understand the consequence, which creates confidence and encourages experimentation.

The story behind PocketEncode is really about removing friction from a repetitive creative task. Whether someone is preparing a video for upload, sharing a draft, or trimming a file for handoff, the tool behaves like a thoughtful assistant. It does not add ceremony. It gives people a clean path from source video to smaller, usable output. That is the kind of utility design that feels invisible in the best possible way.

**Thumbnail brief**  
- Render the actual PocketEncode UI as a clean browser product plate
- Show compression controls, size target, and output state clearly
- Keep it crisp, centered, and product-first
- Use a dark, premium background with a clear sense of control

**Hero brief**  
- Show a creator or designer using the product on a laptop
- Include the UI floating or reflected as a polished output workflow
- Make it feel like a real marketing hero, not a mock screenshot
- Add subtle motion, depth, and a working-session atmosphere

## 2) DriveLink Direct

**Product positioning**  
Google Drive links converted into direct-download URLs.

**Short Description**  
A one-click utility that turns Google Drive share links into direct URLs, making asset handoffs faster and less error-prone.

**Long Description**  
DriveLink Direct started with a very familiar problem: a share link that is technically valid, but practically inconvenient. I designed the tool to collapse that friction into one clear action. The interface only asks the user to paste a Drive link, generate the direct URL, and copy it. That simplicity is the product. The challenge was to keep the experience light while still making it feel dependable and intentional.

From a UX perspective, the key was trust. If a user is converting links for a deck, a website, or a handoff, the result has to feel obvious and reliable. I made the input feedback direct, the error state clear, and the copy interaction immediate so the page feels like a tool built for production use. There is no extra ceremony, because the task itself is already small and specific.

What I like about DriveLink Direct is that it quietly improves a workflow people already have. It does not try to be a bigger product than it is. Instead, it takes one repetitive moment and makes it faster, cleaner, and more predictable. That is often the most useful kind of UX work: removing enough friction that a small task stops being annoying and starts feeling effortless.

**Thumbnail brief**  
- Show the actual tool UI with the input, generated direct link, and copy button visible
- Make the conversion result the hero of the frame
- Include the “paste → generate → copy” flow visually
- Keep it compact and useful, like a utility app card

**Hero brief**  
- Show a person at a desk preparing assets for a site or presentation
- Include browser windows, folders, and a feeling of handoff work
- Make the direct link workflow feel quick, practical, and trustworthy
- Add some real-world creative-office energy without losing the product focus

## 3) WebP Forge

**Product positioning**  
In-browser image conversion with quality, scale, blur, and size comparison controls.

**Short Description**  
An in-browser image converter that turns PNGs, JPGs, SVGs, and more into WebP with live optimization controls.

**Long Description**  
WebP Forge sits right in the sweet spot between power and clarity. People who need image conversion usually want to move fast, but they still need to understand the tradeoffs between quality, size, and output performance. I designed the interface to make those tradeoffs visible without making the experience feel technical or heavy. The tool should feel like a guided workspace, not a settings panel.

The UX idea was to make every important variable easy to inspect. Quality, scale, blur, metadata stripping, and before/after preview all help the user see what changes matter. Instead of hiding the conversion process, the interface puts it in front of the user in a calm and legible way. That turns file optimization into a process of informed choice rather than blind trial and error.

As a designer, I see WebP Forge as a practical craft tool. It helps users preserve what matters while trimming what does not. The preview and comparison states create confidence, and the output feedback makes the tool feel fast and credible. That is the kind of product I like building: one that takes a repetitive technical task and gives it shape, feedback, and a little bit of pleasure.

**Thumbnail brief**  
- Show the WebP Forge UI with the original and output previews visible
- Include quality, scale, and size-comparison cues
- Make the conversion flow feel modern and precise
- Keep the frame balanced so it reads as a product card, not a busy dashboard

**Hero brief**  
- Show a designer or creator working with image assets on a laptop
- Surround the scene with export-ready creative materials
- Make the conversion story feel like a real workflow moment
- Use polished lighting and subtle depth so the product feels marketing-ready

## 4) ToneSplit Posterizer

**Product positioning**  
Photo posterization and tonal layer separation for print workflows.

**Short Description**  
A posterization tool that turns photos into tonal layers, making print-ready separations easy to export and reuse.

**Long Description**  
ToneSplit Posterizer is one of those tools where the interface has to teach the process as much as it performs it. When a user uploads a photo and watches it split into tonal layers, they are not just processing an image; they are learning how that image can become a new kind of asset. I designed the experience to keep that transformation clear, predictable, and satisfying.

The UX challenge was to make the controls understandable without flattening the creative nature of the tool. The user needs to adjust shades, contrast, brightness, and smoothness, but they also need to understand how each choice affects the output. That is why the preview, layer separation, and export stages matter so much. They create a sense of progress and help the user move from curiosity to confidence.

ToneSplit is best understood as a bridge between digital craft and physical output. It supports print workflows, stencil making, and experimental layering, but it also feels satisfying because the result is immediate and tangible. The product celebrates the transformation instead of hiding it. That clarity is what makes the tool feel designed, not just functional.

**Thumbnail brief**  
- Show the posterizer UI with a strong result preview and visible tonal separation
- Include the layer controls or settings panel as part of the story
- Let the before/after transformation feel obvious
- Keep it clean, bold, and visually instructive

**Hero brief**  
- Show a creative person using the tool in a studio or print-oriented environment
- Include paper, stencil, or layered-output cues nearby
- Make the workflow feel hands-on and useful for physical production
- Add stronger texture and a slightly more artistic mood than the thumbnail
